# guards — 各ガードの「なぜ」と型の選び方

ハーネスの心臓部。散文の規約を、ツール呼び出し時に機械的に効く経路（hook）へ昇格させる。原本は [`templates/.claude/hooks/`](../../templates/.claude/hooks/)。

## ガードの3つの型

| 型 | 何をする | いつ使う | 安全方向 |
|---|---|---|---|
| **block** | 条件に当たれば拒否（deny） | 違反を機械的に確実判定でき、止めるべきとき | 取りこぼし(false negative)が**事故**になる → 確実判定できる対象だけに絞る |
| **reminder** | 止めずに注意文を文脈へ注入（allow + context） | 違反かどうか機械判定できないとき | 判断材料を渡すだけ。正規用途を殺さない |
| **allow** | 通常止まる操作を条件付きで先回り許可 | 規約上やってよい操作が毎回ブロックされるとき | 取りこぼしは確認プロンプトに倒れる＝無害 |

新ガードを足すときの3問: ①機械的に確実判定できる? → Noなら reminder。 ②取りこぼしは事故か確認で済むか? ③相手は悪意ある回避か善意のミスか? → 善意なら素朴な検出で十分。**「見逃しも誤検知も多い」ガードは作らない**（邪魔なだけ）。

## このワークスペースのガード

### scope-guard（block / PreToolUse）
担当リポ外への**書き込み**（Write/Edit/危険なbash）を拒否する。[ADR-0005](../decisions/0005-one-session-one-repo.md) の物理的裏打ち。判定は「編集先パスが、このリポのルート配下か、`cross-repo-allowlist` に列挙された隣接リポか」だけ — **パスは機械的に確実判定できる**ので block が適切。

### decision-reminder（reminder / PreToolUse）
`docs/decisions/` 配下を書くときに、採録基準（横断的か/コードで復元できないか/薄く保て）を注意文として注入する。「これは横断知識か、その場限りか」は**機械判定できない**ので block にせず reminder にする典型（[ADR-0009相当の薄い記録運用]）。

### memory-guard（reminder / PreToolUse）
個人メモリへの書き込み時に「恒久・横断の知識なら docs-hub へ振り直せ」を注入する。エージェント本体には「学んだら記憶せよ」という逆向きの常時指示があり得るので、書き込み直前に減速帯（speed-bump）を置く。これも判定不能なので reminder。

### lint / typecheck（PostToolUse=軽量, Stop=フル）
編集直後は重大・高速な検査だけ、ターン終了時にフル lint + typecheck。修正試行後の再検査は止めず通知に倒し、直せないエラーで無限ループさせない。ツール未導入などの環境差では黙って通す。

### core 純粋性チェック（rule + lint）
`core` から `react`/DOM/`Math.random`/`Date` への依存を検出（[ADR-0002](../decisions/0002-pure-core-engine.md)）。静的に判定できるので将来 lint ルール化（block 寄り）。

## 2段構え

重要ルールは ①必読の場所（各リポ CLAUDE.md にインライン）＋ ②hookで強制、の2段で効かせる。リンクだけ・散文だけでは漏れる。
