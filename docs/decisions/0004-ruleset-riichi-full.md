# ADR-0004: 対象ルールは日本リーチ麻雀（フル）。役・点数は段階的に実装する

- ステータス: Accepted
- 日付: 2026-06-22
- 関連: [yaku-scoring-design](../design/yaku-scoring-design.md), [game-flow-design](../design/game-flow-design.md)

## 決定

ゲームの対象ルールは **4人打ち・日本リーチ麻雀のフルルール**とする。これは最終到達点であり、実装は段階的に行う。**段階化は「やらない」決定ではなく「実装順序」の決定**である。

採用する標準（既定値。変更時はADR追加）:
- 4人打ち東南戦（半荘）。25000点持ち / 30000点返し。
- 喰いタンあり・後付けあり（一般的なネット麻雀の既定）。
- 赤ドラ: 各色5に1枚（赤5、計3枚）。
- ダブロン・頭ハネ等の同時和了規定、流局時の聴牌料、連荘ルールは [game-flow-design](../design/game-flow-design.md) で明文化。

## 段階（フェーズ）

| Phase | 範囲 | 検証 |
|---|---|---|
| 1 | 牌モデル・手牌表現・牌山・配牌・ツモ/打牌・**和了形判定（4面子1雀頭・七対子・国士）**・聴牌/待ち列挙 | ユニットテスト（既知手牌） |
| 2 | **役判定**（リーチ/ツモ/平和/断么九/役牌/一盃口/三色/一気通貫/対々/七対子/混一/清一/国士/etc）・翻数 | 役ごとのテスト表 |
| 3 | **符計算・点数計算**（親/子、ツモ/ロン、満貫〜役満、切り上げ）・本場/供託 | 点数早見表との突合 |
| 4 | 局進行の完全実装（鳴き・リーチ宣言・フリテン・流局・連荘・順位確定） | シナリオ/E2E |

## 背景

リーチ麻雀は役・符・特殊規定が多く、一度に正しく作るのは困難。和了形判定という土台が全役・点数計算の前提になるため、土台→役→点数→進行の順で、各段を独立テストしながら積む。「フルを目指す」ことを先に確定させ、簡易版に逃げて手戻りすることを防ぐ。

## 帰結

- **良い**: 各フェーズが独立にテスト可能。役や点数の取りこぼしはADRではなくテスト表で管理し、決定記録を薄く保てる（[ADR-0009相当の運用]はテスト側）。
- **悪い / コスト**: Phase 2-3 は規則量が多く実装が重い。→ ルールの根拠（点数表・符の数え方）は `docs/design/yaku-scoring-design.md` と `investigation/` に集約し、実装が参照する単一の表を持つ。
- ローカルルールのバリエーション（喰い変え、流し満貫、人和の扱い等）は既定値を design に書き、変更はADRで上書きする。

## 検討した代替案

- **簡易リーチ（主要役のみ・符簡略）**: 早く動くが、符計算や特殊役を後付けすると和了/点数の中核を作り直すことになる。ユーザー要件が「フル」なので却下。土台設計はフルを前提にしておき、未実装役は「テスト未追加」として扱う。
