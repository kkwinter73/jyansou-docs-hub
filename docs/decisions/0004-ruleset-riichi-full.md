# ADR-0004: 対象ルールは日本リーチ麻雀（フル）。役・点数は段階的に実装する

- ステータス: Accepted
- 日付: 2026-06-22（2026-06-23 粒度拡充）
- 関連: [yaku-scoring-design](../design/yaku-scoring-design.md) / [game-flow-design](../design/game-flow-design.md) / [core-domain-design](../design/core-domain-design.md)

## コンテキストと問題提起

ゲームの対象ルールを確定する必要がある。リーチ麻雀は役・符・特殊規定が多く、一度に正しく作るのは困難で、簡易版から始めると土台（和了/点数の中核）を後で作り直すリスクがある。「どこまでを目標とし、どの順で実装するか」を最初に決める。

## 決定要因

- ユーザー要件が「日本リーチ麻雀のフル」であること
- 役・点数の取りこぼしを安全に管理でき、手戻りを最小化できること
- 各段が独立にテスト可能で、正しさを積み上げられること
- ローカルルールの差異を後から上書きできること

## 決定結果

**4人打ち・日本リーチ麻雀のフルルール**を到達目標とする。これは最終形であり、実装は段階的に行う。**段階化は「やらない」決定ではなく「実装順序」の決定**である。

### 採用する標準（既定値。変更時はADRで上書き）

- **根拠**: 一般的なネット麻雀の慣行に合わせ、判断を一意にする。
- **影響**: 4人打ち東南戦（半荘）／25000点持ち・30000点返し／喰いタンあり・後付けあり／赤ドラ各色1枚（計3）／切り上げ満貫なし・数え役満あり／頭ハネあり。詳細は [game-flow-design](../design/game-flow-design.md)・[yaku-scoring-design](../design/yaku-scoring-design.md)。

### 実装フェーズ

- **根拠**: 和了形判定が全役・点数の前提なので、土台→役→点数→進行の順に積む。各段を独立テストで固定する。
- **影響**:

| Phase | 範囲 | 状態 |
|---|---|---|
| 1 | 牌・手牌・牌山・配牌・和了形判定（標準形/七対子/国士）・聴牌/待ち列挙・seed可能PRNG | 実装済み |
| 2 | 役判定（通常役一式＋役満）・ドラ集計・高点法 | 実装済み |
| 3 | 符計算・点数計算・支払い・本場・供託 | 実装済み |
| 4a | 局進行（鳴きなし完全ループ）・リーチ・フリテン・一発・流局・連荘・順位 | 実装済み |
| 4b | 鳴き（チー/ポン/カン）・カンのドラ/嶺上・途中流局・流し満貫 | 未実装 |

## 利点

- 「フルを目指す」ことを先に確定させ、簡易版に逃げて中核を作り直す手戻りを防げる。
- 各フェーズが独立にテスト可能（実績: 役14件・点数18件・局65/67件のテスト群）。
- 役・点数の取りこぼしは ADR ではなく `core/tests/` のテスト表で管理でき、決定記録を薄く保てる。

## トレードオフ

- Phase 2-3 は規則量が多く実装が重い。→ ルールの根拠（点数早見表・符の数え方）を [yaku-scoring-design](../design/yaku-scoring-design.md) と `investigation/` に集約し、実装が参照する単一の表を持つ。
- フル前提の土台設計は初期コストが高い。→ ただし簡易版から作り直すコストを回避できるため正味では有利。

## 不採用の選択肢と根拠

| 選択肢 | 不採用理由 |
|---|---|
| **簡易リーチ（主要役のみ・符簡略）** | 早く動くが、符計算や特殊役を後付けすると和了/点数の中核を作り直すことになる。要件が「フル」なので不採用。土台はフル前提で設計し、未実装役は「テスト未追加」として扱う。 |
| **三人麻雀・他ローカルルール主体** | 要件は4人打ちリーチ。ローカル差は既定値＋ADR上書きで吸収する。 |
