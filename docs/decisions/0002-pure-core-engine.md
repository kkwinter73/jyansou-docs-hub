# ADR-0002: ゲームロジックを純粋・決定論的な `core` リポに分離する

- ステータス: Accepted
- 日付: 2026-06-22
- 関連: [ADR-0006](0006-immutable-state-reducer.md), [ADR-0007](0007-seedable-prng.md), [core-domain-design](../design/core-domain-design.md), [game-flow-design](../design/game-flow-design.md)

## 決定

麻雀のルール（牌・手牌・和了判定・聴牌/待ち・役・符・点数・局進行）は、`core` リポ内の **副作用を持たない純粋関数**として実装する。DOM・React・タイマー・`Math.random`・`Date.now`・ネットワーク・ストレージへの依存を `core` から禁止する。

## 背景

ユーザー要件は「シングルプレイ（対CPU）を先に作るが、オンライン対戦も見据える」。オンライン対戦では**サーバが権威**を持ち、同じルール判定をサーバ・クライアント双方で走らせたい（不正検知・楽観的UI）。ロジックがUIや乱数・時刻に依存していると、サーバへ移植できず二重実装になる。

## 帰結

- **良い**: `core` は Node（サーバ）でもブラウザでも同一に動く。入力が同じなら出力も同じ（参照透過）なので、ユニットテストが容易で、リプレイ・牌譜検証が可能。
- **制約（強制したい不変条件）**:
  - 乱数は引数で受け取った seed 可能PRNGからのみ取得する（[ADR-0007](0007-seedable-prng.md)）。`core` 内で `Math.random()` を呼ばない。
  - 時刻・タイマーは `core` の外（`web`/`server`）が持つ。`core` はターン制の状態遷移のみを表現する。
  - 状態は不変データ + 遷移関数で表す（[ADR-0006](0006-immutable-state-reducer.md)）。
- **悪い / コスト**: UI都合の「ちょっとした状態」を `core` に置けず、`web` 側に持つ必要がある。境界設計の規律コストがかかる。→ 境界は [architecture](../design/architecture.md) に明文化し、lint/rule で `core` の禁止import（react等）を検出する。

## 検討した代替案

- **`web` 内にロジックを書く**: 初速は速いが、オンライン化でサーバへ切り出せず全面再実装になる。「両方見据える」という要件に反するため却下。
