# ADR-0006: ゲーム状態は不変データ + 遷移関数（reducer）で表現する

- ステータス: Accepted
- 日付: 2026-06-22（2026-06-23 粒度拡充）
- 関連: [ADR-0002](0002-pure-core-engine.md) / [ADR-0007](0007-seedable-prng.md) / [game-flow-design](../design/game-flow-design.md)

## コンテキストと問題提起

麻雀は「ツモ・打牌・チー/ポン/カン・リーチ宣言・和了・流局」という離散イベントで進む典型的な状態機械である。この進行を `core`（[ADR-0002](0002-pure-core-engine.md)）でどう表現するかを決める。

可変状態を各所で書き換える設計は、(1) どこで状態が壊れたか追えない、(2) UIの楽観更新とサーバ確定のズレを扱えない、(3) リプレイ/牌譜を再現できない、という問題を生む。

## 決定要因

- 決定論（同じ入力列から同じ最終状態）が成り立つこと
- 各遷移を副作用なしで単体テストできること
- UIの楽観更新とサーバ確定の差し替えを扱えること
- 非合法操作を安全に表現できること

## 決定結果

### 状態は不変（immutable）

- **根拠**: 破壊的変更を排し、「ある時点の状態」を常に保持できるようにする。
- **影響**: `GameState` は不変。遷移ごとに新しい状態を返す。実装では `apply` の冒頭で `structuredClone` により複製し、複製側だけを変更する（`core/src/game.ts`）。

### 遷移は `apply(state, action) -> { state, events }`

- **根拠**: プレイヤー操作・自動進行をすべて明示的な `Action`（データ）で表し、状態遷移を1つの純粋関数に集約する。`events` で「何が起きたか」を返し、UIやログが参照できる。
- **影響**: `Action` は `draw` / `discard(+riichi)` / `tsumo` / `ron` / `pass`（鳴きは Phase 4b）。`legalActions(state, seat)` が合法手だけを列挙し、UI/CPUはそこから選ぶ。

### 非合法操作は例外を投げない

- **根拠**: 例外は制御フローを乱し、UI/CPUが扱いにくい。
- **影響**: 非合法な `Action` は状態を変えず `events: [{ type: 'illegal', reason }]` を返す。合法性は `legalActions` が保証する。

### 乱数も状態に内包

- **根拠**: 乱数を伴う遷移（牌山生成・ツモ）でも `apply` の純粋性を保つため。
- **影響**: PRNGの状態（`RngState`）を `GameState` に内包して持ち回る（[ADR-0007](0007-seedable-prng.md)）。

## 利点

- **決定論**: `(初期state, actionの列)` から最終stateが一意に再現でき、牌譜リプレイ・サーバ権威化・テストが容易。
- **検証**: 各 `apply` を副作用なしで単体テストできる（実績: ツモ切り/CPU自己対戦で「全点数＋供託＝100000」の保存を多数seedで確認）。
- **UI**: `web` は確定stateを描画するだけ。楽観更新は「予測apply→サーバ確定で置換」で扱える。

## トレードオフ

- 毎遷移で新オブジェクトを作るため素朴には割当が増える。→ 麻雀は1局の遷移数が小さく実用上問題ない。必要なら構造共有（Immer等）を後で局所導入（その時ADR追加）。
- `structuredClone` は標準ライブラリ外のグローバル。→ Node 17+/主要ブラウザで利用可。型は `globals.d.ts` で補完し、lib に DOM/Node を含めない。

## 不採用の選択肢と根拠

| 選択肢 | 不採用理由 |
|---|---|
| **可変な `Game` オブジェクト（メソッドが内部状態を変更）** | 直感的だが決定論・リプレイ・サーバ共有の利点を失う。 |
| **イベントソーシング（イベントのみ保存しstateは都度畳み込み）** | reducer設計の自然な拡張で将来移行可能（`apply` が `events` を返す形にしてある）。現時点は state を直接持つ方が単純。 |
| **例外で非合法を表現** | 制御フローを乱し UI/CPU が扱いづらい。結果型（イベント）で表す（`typescript.md` 規約とも整合）。 |

## 注記

- `apply` の戻り値 `events` は現状 UI/ログ向けの軽量イベント。牌譜の永続化が必要になれば、`Action` 列の保存（イベントソーシング）へ拡張できる。
