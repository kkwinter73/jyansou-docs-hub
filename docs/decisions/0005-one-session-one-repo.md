# ADR-0005: 1セッション = 1担当リポ制を採用する

- ステータス: Accepted
- 日付: 2026-06-22（2026-06-23 粒度拡充）
- 関連: [ADR-0001](0001-multi-repo.md) / [ADR-0008](0008-git-topology.md) / [workflow](../dev-environment/workflow.md) / [guards](../dev-environment/guards.md)

## コンテキストと問題提起

マルチリポ構成（[ADR-0001](0001-multi-repo.md)）では、ある作業のついでに別リポを書き換えると事故が起きる。(1) その別リポの規約・ガード・文脈（CLAUDE.md）が読まれないまま編集される、(2) 担当境界が崩れてレビュー単位・作業履歴が壊れる、(3) 横断SoT（docs-hub）が副作用で書き換わりレビューを通らずに真実源が変わる。

人間にもエージェントにも適用できる「どのリポを、どの範囲で触ってよいか」のプロセス規約が要る。

## 決定要因

- 作業中リポの文脈（CLAUDE.md・rules・hooks）が常に正しくロードされること
- 担当境界が崩れず、レビュー単位とコミット/Issue履歴がリポ単位に揃うこと
- 横断SoTが末端の作業で勝手に書き換わらないこと
- 散文の約束でなく、機械的に強制できること

## 決定結果

1つの作業セッション（エージェント or 人間）は、**1つのリポの担当者**として振る舞う。

### 書き込みは担当リポ内だけ（読み込みは他リポ可）

- **根拠**: 編集対象のリポの規約・ガードが必ず効く状態を保つ。他リポは参照（`../core` 等）するが書き換えない。
- **影響**: 担当外パスへの書き込みは scope-guard hook が拒否する（block型。[guards](../dev-environment/guards.md)）。所有関係のある隣接リポだけ `cross-repo-allowlist` で例外許可する。

### 他リポの変更は Issue で引き継ぐ

- **根拠**: 別リポの変更は、その別リポの文脈を持つセッションで行うべき。
- **影響**: その場で書かず Issue を立てる。横断SoTへの反映可否（決定として残すか等）は docs-hub 側が判断し、末端で記録形式を勝手に作らない（[ADR-0008](0008-git-topology.md) 関連の SoT 保護）。

## 利点

- 作業ディレクトリ・CLAUDE.md・ガード設定が常に一貫する。
- 「Aの担当者が勝手にBを書き換える」責任境界の崩壊を防げる。
- 作業履歴がリポ単位のコミット／Issueに自然に揃う。

## トレードオフ

- 1つの変更が複数リポに波及するとき、1セッションで完結できず引き継ぎコストがかかる。→ 責任分界を守るための意図的なコスト。実例: core の公開API変更→web の追従は、core セッションで Issue を立て、web セッションで対応。
- 運用上の規約なので、機械的強制（hook）と併用しないと破られうる。→ scope-guard で裏打ち。

## 不採用の選択肢と根拠

| 選択肢 | 不採用理由 |
|---|---|
| **境界なし（どこでも編集可）** | 個人開発では一見楽だが、ロジック/UIの責任分界が崩れ、`core` の純粋性（[ADR-0002](0002-pure-core-engine.md)）や docs-hub の SoT が守られなくなる。 |
| **散文の規約だけ（hookなし）** | 「書いただけのルール」は守られない。必読の場所への明記＋hookの2段構えが要る（[guards](../dev-environment/guards.md)）。 |

## Claude Code ルール連携

- `templates/.claude/hooks/scope-guard.mjs` — 担当リポ外への書き込みを拒否（block型）。
- `<repo>/.claude/cross-repo-allowlist` — 越境書き込みの例外を明示。
- `docs/dev-environment/workflow.md` — ピック→計画→実装→コミット→引き継ぎの定型フロー。

## 注記

- scope-guard は編集ツール（Write/Edit 等）の `file_path` を対象とする。Bash コマンド文字列からの書き込み検出は false-negative が多く「効かないガード」になるため、あえて対象にしない（[guards](../dev-environment/guards.md) 原則）。
